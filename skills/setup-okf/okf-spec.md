# OKF v0.1 Specification

This repo uses OKF (Open Knowledge Format) v0.1. The OKF bundle lives under `docs/app/`.

## Bundle structure

```
docs/app/
├── index.md        # auto-generated; do not hand-edit
├── log.md          # audit trail of changes
├── concepts/       # glossary and domain terms
├── decisions/      # design decisions
├── systems/        # systems, services, components
├── datasets/       # logical datasets (data projects)
├── tables/         # physical tables (data projects)
├── pipelines/      # data pipelines (data projects)
└── playbooks/      # runbooks and procedures
```

Concepts may be grouped into context subdirectories, e.g. `concepts/<feature>/<term>.md`. Each context directory has its own `index.md`, auto-generated by `okf-tool`.

## Concept files

Every concept is a markdown file with YAML frontmatter:

```markdown
---
type: Concept
title: Canonical Term
description: One-line definition.
timestamp: 2026-06-30T00:00:00Z
---

# Definition

# Examples

# Related

- [Other term](/concepts/<context>/<other>.md)
```

### Required frontmatter

- `type` — one of: Concept, Decision, System, Dataset, Table, Pipeline, Playbook.
- `title` — canonical human-readable name.
- `description` — one-line summary.
- `timestamp` — ISO 8601 creation/last-significant-update time.

### Reserved filenames

- `index.md` — auto-generated directory index.
- `log.md` — change log at bundle root.

## Cross-links

Use bundle-relative paths from `docs/app/`:

```markdown
- [Customer](/concepts/orders/customer.md)
- [Use PostgreSQL](/decisions/0001-use-postgres.md)
```

Always include the `.md` extension.

## Tools

The `scripts/okf/` package in this repo provides `okf-tool`:

```bash
python -m scripts.okf.cli index      # regenerate all index.md files
python -m scripts.okf.cli validate   # check frontmatter, links, orphans
python -m scripts.okf.cli log "..."  # append to docs/app/log.md
python -m scripts.okf.cli viz        # generate docs/viz.html graph
python -m scripts.okf.cli show path  # print a concept file
```

Always run `okf-tool index` after adding or changing a concept. Run `okf-tool validate` before finishing a batch of edits to catch broken links or missing frontmatter.
